このサイトについて

このサイトは、OpenPNE3.8系から新たに追加された、スマートフォンAPIの仕様についてまとめたサイトです。

左のリストから、見たいエントリポイントの仕様を選択してください。


【共通仕様】リクエスト

スマートフォンAPIには、予め共通仕様のAPIリクエストが定められています。

以下のリクエストは必須項目となっており、このフィールドが満たされていない場合は一部の例外を除きAPIが動きません。

パラメータ 値 説明
apiKey *必須 string APIにリクエストする為のAPI認証キーをセットします。
APIキーはリクエストするメンバー毎にユニークなIDとなっています。
また、APIキーはpc_frontendのHTMLコードの中にあるopenpneというjavascript変数にセットされています。



【共通仕様】サンプルリクエスト

apiKeyのセット方法について

$ curl http://example.com/sns/api.php/activity/search.json?target=community&target_id=1&count=20&apiKey=5ad9d5a6a11c5f13ca5901a7c6e1d89ea991dc195666de393a61c808c61d19a5

と指定することによりAPIリクエストをすることができます。


【共通仕様】レスポンス

スマートフォンAPIには、予め共通のレスポンスフィールドの元で返されるようになっています。

フィールド名 説明
status APIが正常に動いたかを出力します。通常成功した場合はsuccess、リクエストに失敗した場合はerrorを返すようになっています。
data APIリクエストした際に特定のデータを返すAPIについては、そのデータをこのフィールドの中に配列で返されるようになっています。
message APIリクエスト時に特定のメッセージを返すAPIについては、そのメッセージをこのフィールドの中に返されるようになっています。

【共通仕様】レスポンスのメンバー情報

レスポンスの中でもメンバー情報を取り扱うものは、以下のフィールドの共通仕様に合わせるべきです。

フィールド名 説明
member.id メンバーIDを返します。
member.profile_image メンバーのプロフィール画像を絶対URLで返します。プロフィール画像が登録されていない場合は、no_image.gifを絶対URLで返します。
member.name メンバーのニックネームを返します。
member.friend 自分とフレンドリンクであればtrue、そうでなければfalseを返します。
member.blocking 自分がアクセスブロックしているメンバーであればtrue、そうでなければfalseを返します。
member.self このメンバー情報は自分自身のものであればture、そうでなければfalseを返します。
member.friends_count メンバーのフレンドリンクの数を返します
member.self_introduction メンバーのプロフィールにある自己紹介を返します。プロフィール項目に自己紹介が設定されていない場合はfalseを返します。
