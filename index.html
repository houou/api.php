このサイトについて

このサイトは、OpenPNE3.8系から新たに追加された、スマートフォンAPIの仕様についてまとめたサイトです。

左のリストから、見たいエントリポイントの仕様を選択してください。


【共通仕様】リクエスト

スマートフォンAPIには、予め共通仕様のAPIリクエストが定められています。

以下のリクエストは必須項目となっており、このフィールドが満たされていない場合は一部の例外を除きAPIが動きません。

パラメータ 値 説明
apiKey string APIにリクエストする為のAPI認証キーをセットします。
APIキーはリクエストするメンバー毎にユニークなIDとなっています。
また、APIキーはpc_frontendのHTMLコードの中にあるopenpneというjavascript変数にセットされています。


【共通仕様】レスポンス

スマートフォンAPIには、予め共通のレスポンスフィールドの元で返されるようになっています。

フィールド名 説明
status APIが正常に動いたかを出力します。通常成功した場合はsuccess、リクエストに失敗した場合はerrorを返すようになっています。
data APIリクエストした際に特定のデータを返すAPIについては、そのデータをこのフィールドの中に配列で返されるようになっています。
message APIリクエスト時に特定のメッセージを返すAPIについては、そのメッセージをこのフィールドの中に返されるようになっています。
